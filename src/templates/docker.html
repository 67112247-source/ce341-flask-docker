สรุป Docker

Docker คือเครื่องมือสำหรับสร้างและจัดการ container ซึ่งเป็นสภาพแวดล้อมเล็ก ๆ ที่รวมทั้งโค้ดและ dependency เอาไว้ด้วยกัน ทำให้สามารถย้ายแอปพลิเคชันไปรันบนเครื่องอื่นได้ง่ายขึ้น โดยไม่ต้องกังวลเรื่องการติดตั้งแพ็กเกจซ้ำหลายรอบ

แนวคิดสำคัญของ Docker

  • Image – แม่แบบ (template) ที่เก็บโค้ดและสภาพแวดล้อมเอาไว้
  • Container – instance ของ image ที่กำลังรันอยู่จริง
  • Dockerfile – ไฟล์ที่ใช้บอกขั้นตอนการสร้าง image
ตัวอย่างการสร้าง image และรัน container ด้วยคำสั่ง docker build, docker run และตรวจสอบสถานะด้วย docker ps บน Docker Desktop

ตัวอย่างคำสั่ง Docker ที่ใช้ในงานนี้

  • docker build -t virada/ce314-flask-web:v1 . – สร้าง image จาก Dockerfile
  • docker run -p 5000:5000 virada/ce314-flask-web:v1 – รัน container และเปิดใช้งานผ่าน port 5000
  • docker images – แสดงรายการ image ที่มีอยู่ในเครื่อง
  • docker ps – แสดง container ที่กำลังรันอยู่
  • docker push virada/ce314-flask-web:v1 – ส่ง image ขึ้น Docker Hub

ในโปรเจกต์นี้ เราใช้ Docker เพื่อสร้าง image ของเว็บที่เขียนด้วย Flask จากนั้นสามารถนำ image นี้ไปรันบนเครื่องอื่น หรือบนเซิร์ฟเวอร์ได้สะดวก โดยไม่ต้องตั้งค่า Python และ dependency ใหม่ทุกครั้ง
