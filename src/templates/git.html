สรุป Git & GitHub

Git คือระบบจัดการเวอร์ชันของไฟล์ (Version Control) ใช้เก็บประวัติการแก้ไขโค้ด สามารถย้อนกลับไปดูเวอร์ชันก่อนหน้าได้ และช่วยให้การทำงานร่วมกันเป็นทีมเป็นระเบียบมากขึ้น

GitHub เป็นบริการบน cloud สำหรับเก็บ repository ของ Git ทำให้เราสามารถแชร์โค้ด ดูประวัติการแก้ไข และทำงานร่วมกันผ่านอินเทอร์เน็ตได้สะดวก

ตัวอย่างคำสั่ง Git ที่ใช้บ่อย

  • git init – เริ่มต้นสร้าง repository ใหม่ในโฟลเดอร์ปัจจุบัน
  • git status – ตรวจสอบสถานะไฟล์ที่ถูกแก้ไข
  • git add . – เพิ่มไฟล์ทั้งหมดเตรียมสำหรับ commit
  • git commit -m "ข้อความ" – บันทึกการเปลี่ยนแปลงลงในประวัติ
  • git remote add origin <url> – เชื่อม repo ในเครื่องกับ GitHub
  • git push -u origin main – ส่งโค้ดจากเครื่องขึ้นไปยัง GitHub

ในงานนี้ เราใช้ Git และ GitHub เพื่อเก็บโค้ด Flask และ Dockerfile ทำให้ง่ายต่อการส่งงานและตรวจสอบประวัติการพัฒนาโปรเจกต์
